# CFY 实验数据并发管理系统 (Backend)

本后端服务基于 FastAPI 构建，专为处理大规模示波器数据（CSV）、多通道智能对比以及高并发下的极速响应而设计。由于此前引入了“微服务算力隔离架构”，系统现在的启动步骤被划分为两个核心部分。

---

## 一、环境与依赖安装

请确保当前终端已经激活了虚拟环境 `.venv`。如果缺失依赖模块，可执行：
```bash
pip install -r requirements.txt
# 补充新架构依赖库
pip install httpx requests
```

---

## 二、如何将系统“跑”起来？（重要）

为了实现数据清洗算力与 API 主网关的最佳隔离，**请强制按以下双节点模式启动**（推荐打开两个独立的终端分别运行）：

### 👉 终端 1：启动核心计算微服务 (Worker Node)
由于 `services/signal_processing.py` 中的 Numpy 与 Pandas 会有 GIL 阻塞瓶颈，这部分被单独隔离进了子服务中。
在项目根目录运行：
```bash
python worker_main.py
```
> **成功标志**：看到提示 `INFO: 内部计算网络微服务启动监听 8001 端口...`

### 👉 终端 2：启动主应用架构 (API Gateway)
当计算后台挂起后，即可唤起管理所有 Web 流量防线的主服务（也是前端直接对接的服务）：
```bash
python main.py
```
或者使用热更新模式运行：
```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```
> **成功标志**：首次启动后会看到生成 `admin/123456` 默认总管账号的声明，并在监听 `0.0.0.0:8000`。

## 三、调试与压测指南

### 1. 监控拦截 (日志中心)
主系统已集成内置的系统滚动监控，您不需要再在控制台翻找报错：
- 所有中间件流量出入库及崩溃异常，现已被实时定向至根目录下的 `logs/app.log`，调试查错请直接查阅此文件。

### 2. 真实硬件极速上报仿真引擎
在确保前后端、微服务链路跑通后，您可随时向 `8000` 主网关进行轰炸测试：
```bash
python tests/stress_tester.py
```
该引擎会自动骗取 Auth 凭证并多线程携带假数据重炮轰击您的接口，并输出并发量、阻断数及平均毫秒流速报告。
