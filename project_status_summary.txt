# 实验数据管理系统后端 (FastAPI) 当前架构与功能盘点

## 1. 项目全局概况
- **主架构**：FastAPI + Uvicorn (高性能异步处理机制)。
- **持久化层**：SQLAlchemy 2.0 (为保障跨端独立全栈联调的高可用性，目前配置临时平滑切换为主流无状态单体文件引擎 SQLite `cfy_exp.db`，可随时通过更改 URL 切回 MySQL 8.0)。
- **安全与鉴权机制（全新重构）**：自主接管了高版本 Python 3.13 兼容性问题，移除了过时引发依赖级崩溃的 `passlib`，安全重定向接入原生强加密级别 `bcrypt`，并且全链路打通了基于 OAuth2 和 JWT Token 的多角色非对称操作权限验证。

## 2. 当前系统架构拓扑与层级职责

当前后端已演进至高度解耦的**四层全套流转架构模型**（`Subject` -> `Group` -> `Project` -> `ExperimentData`）：

### 2.1 路由总线与全自动挂载中枢
- **`api/project_api.py` (立项与查询检索中枢)**：
  - 构建了防呆倒查级的关联“兜底寻根”引擎：当前端未下属或者由于新建隔离沙箱发出占位符号建项时，将动态探测兜底组环境；若库位空虚，则自动级联静默生产 `Subject` 及 `Group` 进行安全挂靠从而完美避规外键崩塌。
  - **历史图库动态点播重现**：新加入了 `/api/records/{record_id}/waveform` 以及独立明细透传节点。利用本地存储的 `.csv` 或者离线数组即时读取并调用引擎做飞速清洗后传输回溯重绘所需的底层大矩阵；同时通过保留的 `轻量级检索` 函数脱水提取记录日志用于缩略型看板。
- **`api/upload_api.py` (超压上传接收管道)**：
  - **严密对齐大体积表单联调**：深度打通并对齐了前端混合形态 `FormData` 表述层质检规则（Pydantic），顺畅解决诸如前端甩来如 `"PROJ-DEFAULT-000"` 字符串主键强转所引发的报错；利用 `Optional[str]` 解析方案，包容并提取所有元散件，后置收拢生成 `config_json` 归档块。
  - **智能策略双轨解析**：允许直接接收端侧已拆解干净的熟数据（跳过磁盘直击内存），抑或利用备用通道读取物理 `.csv` 落盘留痕清洗。

### 2.2 信号算法核心引擎 (`services/signal_processing.py`)
整个数据台最硬核的数学计算中枢。
- **抗造防穿孔解析 (Robust Fallback)**：对直接上传处理时，根绝了传统的 `skiprows=7` 暴力硬装引发的崩溃。引入 `utf-8-sig` 并智能动态扫描寻址含 “Time” 数据头部，向下泛化兼容了含有 “Voltage” 或各号通道阵列的矩阵群。
- **全内存光速处理板 (High-Performance Numpy 体系)**：调用 `clean_oscilloscope_arrays` 方法全态承接高密度二维数据，倒推计算差值拟合基点频带 `fs` 参数。
- **纯净级零相位偏移滤波**：通过奈奎斯特验证注入 `scipy.signal` 的 4 阶低通 Butterworth 滤波器参数边界，启动关键双向回压 `filtfilt` 机制实现极致丝滑的信号消噪。完全抹平了传统滤波器导致的信号整体顺延时间位宽错开效应，保证原始高振幅与低频完美复刻。

### 2.3 极限并发与系统守护 (`BackgroundTasks`)
- **后台毫秒极速放行**：引入 FastAPI 的 `BackgroundTasks`。在 `upload_experiment_data` 中完成至关重要的算力交割后，立刻回弹图表波形结果给客户看。而将极其耗时的网卡写入物理磁盘以及 SQL `Session.commit()` 等写重脏活推进到子线程执行不挤占前端资源通道（真正意义上的实验异步隔离设计）。

## 3. 后续开发与演进空间
